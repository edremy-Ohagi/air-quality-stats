# AirQualityStats - 城市空气质量信息管理系统

基于 C++ 的命令行空气质量数据查询与统计系统，使用面向对象分层模型与 RAII 内存管理实现对大规模 CSV 数据的多维度分析。

## 功能特性

- 按城市和日期精准查询详细空气质量数据（AQI、PM2.5、PM10、SO₂、NO₂、CO、O₃）
- 查找某城市在指定年份中空气质量最好的月份，并列出当月最优的 10 天
- 在指定时间范围内，找出 AQI 平均值最低（空气最好）的城市
- 统计某城市指定月份各污染物的平均浓度，并指出该月的主要污染物

## 架构设计

项目采用 **面向对象组合（Composition）** 分层架构，底层独立数据类逐层聚合为完整记录，再由 RAII 容器统一管理：

```text
┌─────────┐  ┌─────────┐  ┌─────────┐
│  City   │  │  Date   │  │ Airqlt  │
│ 城市信息 │  │ 日期信息 │  │ 空气质量 │
└────┬────┘  └────┬────┘  └────┬────┘
     │            │            │
     └────────────┼────────────┘
                  ▼
          ┌──────────────┐
          │  Recordinfo  │
          │ （组合记录）  │
          └──────┬───────┘
                 │  聚合为动态数组
                 ▼
          ┌──────────────┐
          │    Loadr     │
          │（RAII 容器）  │
          │ new[] / delete[] │
          └──────────────┘
```

**关键设计决策：**

- **日期压缩编码**：`Date` 类将年、月、日编码为单整型变量（`YMD = day + month * 100 + year * 10000`），使时间区间校验从多步 `if-else` 判定转变为一次性数值比对，显著简化逻辑并提升检索速度
- **RAII 内存管理**：`Loadr` 容器类将动态数组的分配（`new`）与释放（`delete[]`）封装于构造和析构函数内，摒弃"静态超大数组"的传统做法，有效保障内存安全
- **对象组合解耦**：底层划分为 `City`、`Date`、`Airqlt` 等独立数据结构，由 `Recordinfo` 统筹组合，避免"上帝类"的出现
- **零依赖数据解析**：完全基于标准 C++ 文件流与字符串流实现对 50 万行级 CSV 的稳健解析
- **构建自动化**：利用 CMake `configure_file` 机制实现数据文件自动拷贝，提升项目拉取即跑的体验

## 项目亮点

- **完整功能闭环**：覆盖数据加载 - 多维查询 - 统计分析的全流程，具备实用的数据分析工具形态
- **精准内存控制**：预读文件精准统计需求行数后按需分配，无冗余内存浪费
- **高效日期检索**：YMD 编码使海量数据的日期范围过滤仅需一次整数比较
- **纯 STL 实现**：零外部依赖，完全基于标准库完成 CSV 解析与数据处理

## 构建要求

- 支持 C++11 或更高标准的编译器（推荐 GCC 9+ 或 MSVC 2017+）
- CMake 3.15+（推荐）

## 构建方法

### 方法一：使用 CMake（推荐）

```bash
cmake -B build
cmake --build build

# 运行（CMake 已自动将 AQ_1.csv 拷贝至输出目录）
./build/bin/AirQualityStats
```

> **提示**：也可在 VSCode 中安装 `CMake Tools` 扩展直接构建，或在 Visual Studio 中通过 `文件 -> 打开 -> 文件夹` 加载项目运行。

### 方法二：直接使用 g++ 编译

```bash
g++ -std=c++11 -o AirQuality 空气质量统计系统.cpp

# 运行前请手动将 AQ_1.csv 拷贝到当前目录
./AirQuality
```

## 操作说明

程序启动后自动加载数据，进入主菜单交互界面：

| 操作 | 方法 |
| ------ | ------ |
| 按城市和日期查询 | 选择功能 1，输入城市名称和日期 |
| 查找最佳月份 | 选择功能 2，输入城市名称和年份，显示 AQI 最低月份及当月最优 10 天 |
| 查找最佳城市 | 选择功能 3，输入时间范围，显示 AQI 平均值最低的城市 |
| 主要污染物分析 | 选择功能 4，输入城市名称和月份，显示各污染物平均浓度及主要污染物 |
| 退出程序 | 选择功能 5 |

## 数据格式说明

`AQ_1.csv` 前两行为表头，从第三行起为实际数据：

`城市编号, 城市名称, 省份, 日期(YYYY/MM/DD), AQI, 空气质量等级, PM2.5, PM10, SO2, NO2, CO, O3`
