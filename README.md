# 城市空气质量信息管理系统

一个基于 C++ 的命令行空气质量数据查询与统计系统，作为2024冬计算机高级语言课程设计项目开发。
本项目的上传还有另一个目的：练习规范使用GitHub。

## 项目目标

- 综合运用 C++ 面向对象编程（类的封装、动态内存管理、运算符重载等）解决实际数据管理问题
- 实现对大规模 CSV 数据的手动解析与多维度统计分析（按城市、时间、污染物类型）
- 掌握自定义数据结构设计与排序算法（冒泡排序）在实际项目中的应用
- 构建完整的命令行交互程序，涵盖数据加载、查询、统计、输出等完整业务流程
- 规范使用 Git / GitHub 进行版本管理和项目发布

## 技术栈

| 类别 | 具体内容 |
|------|----------|
| **编程语言** | C++11 / C++17 |
| **核心库** | `<fstream>`（文件 I/O）、`<sstream>`（字符串流）、`<iostream>`、`<string>` |
| **编程范式** | 面向对象（OOP）：多类封装、动态内存（`new` / `delete`）、枚举类型 |
| **数据处理** | 手动 CSV 解析、自定义排序（冒泡排序）、多字段聚合统计 |
| **构建工具** | CMake 3.15+ / g++ (MinGW) / MSVC |
| **开发工具** | VSCode + CMake Tools、Visual Studio 2017+ |
| **版本管理** | Git / GitHub |

## 功能特性

- **查询**：按城市和日期查询指定城市的空气质量详情（AQI、PM2.5、PM10、SO₂、NO₂、CO、O₃）
- **最佳月份统计**：查找某城市在指定年份中空气质量最好的月份，并列出当月最优的 10 天
- **最佳城市排名**：在指定时间范围内，找出 AQI 平均值最低的城市
- **主要污染物分析**：统计某城市指定月份各污染物的平均浓度，并指出主要污染物

## 数据文件格式

程序运行依赖于 `AQ_1.csv` 数据文件，需将其放置在**可执行文件的同级目录**下。

文件格式（CSV，前两行为表头，从第三行起为数据）：

```
城市编号, 城市名称, 省份, 日期(YYYY/MM/DD), AQI, 空气质量等级, PM2.5, PM10, SO2, NO2, CO, O3
```

字段说明：

| 字段 | 说明 | 示例 |
|------|------|------|
| 城市编号 | 城市数字编号 | `101` |
| 城市名称 | 城市中文名 | `北京` |
| 省份 | 所属省份/直辖市 | `北京` |
| 日期 | `年/月/日` 格式 | `2020/1/1` |
| AQI | 空气质量指数 | `80` |
| 等级 | 优/良/轻度污染/中度污染/重度污染/严重污染 | `良` |
| PM2.5 | 24小时平均值 (μg/m³) | `55` |
| PM10 | 24小时平均值 (μg/m³) | `72` |
| SO2 | 24小时平均值 (μg/m³) | `8` |
| NO2 | 24小时平均值 (μg/m³) | `45` |
| CO | 24小时平均值 (mg/m³) | `1` |
| O3 | 24小时平均值 (μg/m³) | `60` |

## 构建与运行

### 环境要求

- C++ 编译器，支持 C++11 或更高标准（推荐 GCC 9+ 或 MSVC 2017+）
- CMake 3.15+（推荐，可配合 VSCode / Visual Studio 使用）

### 方式一：CMake（推荐）

适用于 **VSCode（CMake Tools 扩展）** 和 **Visual Studio 2017+**。

```bash
# 在项目根目录执行
cmake -B build
cmake --build build
```

可执行文件位于 `build/bin/`，若源目录存在 `AQ_1.csv`，会自动复制到该目录。

**VSCode 用户**：安装 [CMake Tools](https://marketplace.visualstudio.com/items?itemName=ms-vscode.cmake-tools) 扩展后，直接点击底栏的 **Build** 按钮即可。

**Visual Studio 用户**：使用"打开文件夹"功能直接打开仓库目录，Visual Studio 会自动识别 CMakeLists.txt。

### 方式二：直接编译（g++）

```bash
g++ -std=c++11 -o AirQuality 空气质量统计系统.cpp
```

### 运行

```bash
./build/bin/AirQualityStats   # CMake 构建
./AirQuality                  # 直接编译
```

运行前请确保 `AQ_1.csv` 与可执行文件在同一目录下。

### 使用说明

程序启动后会自动加载数据，随后进入主菜单：

```
####################
        Main Menu
1. Search information by city and time.
2. Show the best month of a city in a specific year.
3. Find the best city in a specific timescale.
4. Show statistics about airpollution by month and city.
5. Exit.
####################
```

- **功能 1**：输入日期（年 月 日）和城市名称，查看该天该城市的完整空气质量数据
- **功能 2**：输入城市名和年份，输出该年空气最优月份及当月最优的 10 个日期
- **功能 3**：输入起止时间范围，输出该时段平均 AQI 最低的城市
- **功能 4**：输入年月和城市名，输出各污染物月均值并指出主要污染物

## 项目亮点

- **纯 STL 实现**：未依赖任何第三方库，完全基于标准 C++ 完成 CSV 解析与数据统计
- **自定义 VLA**：通过 `Loadr` 类封装动态数组，在程序启动时根据 CSV 行数动态分配内存，避免固定数组浪费
- **多维度查询**：支持按城市+日期精确查询、按年份找最优月份（含 Top10 日期排序）、跨城市 AQI 横向比较、按月统计各污染物均值并识别主要污染物
- **CMake 跨平台构建**：提供 CMakeLists.txt，兼容 VSCode、Visual Studio 等主流 IDE，并自动复制数据文件

## 已知限制与优化方向

> 以下为开发过程中留存的改进方向，供参考：

- 数据加载后不可重新加载（未实现覆盖逻辑）
- 排序算法为 O(n²) 冒泡排序，大数据量下性能有限，可改用 `std::sort`
- 城市输入为字符串匹配，未做模糊搜索或拼写纠错
- 数值类型使用 `int`，统计精度受限，可改为 `double`
- 全局文件流对象不利于多次加载，可重构为函数内局部对象
