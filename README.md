# 🏙️ 城市空气质量信息管理系统

一个基于 C++ 的命令行空气质量数据查询与统计系统，作为 2024 冬计算机高级语言课程设计项目开发。
本项目也是规范使用 GitHub 的练习。

---

## ✨ 功能特性与亮点

### 核心功能
- **精准查询**：按城市和日期查询详细的空气质量数据（AQI、PM2.5、PM10、SO₂、NO₂、CO、O₃）。
- **最佳月份统计**：查找某城市在指定年份中空气质量最好的月份，并列出当月最优的 10 天。
- **最佳城市排名**：在指定时间范围内，找出 AQI 平均值最低（空气最好）的城市。
- **主要污染物分析**：统计某城市指定月份各污染物的平均浓度，并指出该月的主要污染物。

### 项目亮点
- **数据压缩与检索优化**：在 `Date` 类设计中，巧妙将年、月、日三项数据转换编码为一个单整型变量（`YMD = day + month * 100 + year * 10000`）。这使得底层的时间区间校验从繁琐的多步 `if-else` 判定，转变为极快的一次性数值比对，显著简化了逻辑并提升了检索速度。
- **基于 RAII 思想的内存管理**：摒弃了传统课设中“静态声明超大数组”的做法。通过预读文件精准统计需求行数，并在 `Loadr` 容器类中严格贯彻 **RAII (资源获取即初始化)** 原则，将数组的动态分配（`new`）与释放（`delete[]`）封装于构造和析构函数内，有效保障了内存安全。
- **清晰的面向对象分层模型**：体现良好的对象聚合（Composition）理念。底层划分为 `City`、`Date`、`Airqlt` 等独立清晰的数据结构，再由 `Recordinfo` 进行统筹组合，避免了“上帝类”的出现，降低了逻辑耦合度。
- **零依赖的纯 STL 数据解析**：未引入任何外部第三方库，完全基于标准 C++ 文件流（`<fstream>`）与字符串流（`<sstream>`）实现了对千行级 CSV 结构文件的稳健解析。
- **高度配置的跨平台构建**：采用现代化的 `CMakeLists.txt` 系统管理项目；主动利用 `configure_file` 机制实现了“从代码编译到同级必要依赖文件（如 `AQ_1.csv`）自动拷贝”的全流程托管，提升了项目拉取即跑的体验。

## 🚀 快速开始

### 1. 环境准备
- 支持 C++11 或更高标准的编译器（推荐 GCC 9+ 或 MSVC 2017+）
- CMake 3.15+（推荐）

### 2. 获取数据文件
程序运行依赖于 `AQ_1.csv` 数据文件。请确保该文件与可执行文件在**同级目录**下。

### 3. 编译与运行

**方式一：使用 CMake（推荐）**

- **VSCode**：安装 `CMake Tools` 扩展，打开项目后点击底部状态栏的 **Build** 即可编译并运行。
- **Visual Studio**：通过 `文件 -> 打开 -> 文件夹` 加载项目，在顶部启动项选中 `AirQualityStats.exe` 点击运行。
- **命令行**：
  ```bash
  cmake -B build
  cmake --build build
  
  # 运行（CMake 已自动将 AQ_1.csv 拷贝至输出目录）
  ./build/bin/AirQualityStats
  ```

**方式二：直接使用 g++ 编译**
```bash
g++ -std=c++11 -o AirQuality 空气质量统计系统.cpp

# 运行前请手动将 AQ_1.csv 拷贝到当前目录
./AirQuality
```

## 🎯 项目教学目标

- 综合运用 C++ 面向对象编程（类的封装、动态内存管理、运算符重载等）解决实际数据管理问题。
- 实现对大规模 CSV 数据的手动解析与多维度统计分析。
- 掌握自定义数据结构设计与排序算法（冒泡排序）在实际项目中的应用。

---

## 🔧 已知限制与优化方向

1. **排序算法**：目前采用 $O(n^2)$ 冒泡排序，大数据量下性能有限，未来可改用 `std::sort`。
2. **交互体验**：城市输入为精准字符串匹配，未实现模糊搜索或拼写纠错。
3. **数据精度**：污染物数值目前使用 `int`，统计平均值时精度受限，可重构为 `double`。
4. **内存管理**：数据加载后暂不支持热更新重新加载。
5. **代码结构**：全局文件流对象不利于多次加载，可重构为局部对象以提升代码鲁棒性。
